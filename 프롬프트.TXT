streamlit을 사용하도록 파이썬 프로젝트 생성한다. 기본적으로 엑셀 파일일 입력 받는다.  엑셀 파일의 F열을 읽는다. F열은 첫 행을 제외하고(HEADER) 2행부터는 영어 단어이다. 해당 영어 단어의 발음기호를 H열에 작성한다. 그리고 영어 단어의 발음 SOUND 파일을 MP3 형식으로 만든다. 파일명은 A열, D열, E열, F열의 텍스트를 "_"로 연결하고 확장자 MP3로 이다. 생성된 파일은 폴더에 저장하고 폴더명은 B열의 텍스트로 한다.


사운드 파일이 이미 존재하는 경우에는 다시 생성하지 않도록 수정한다


발음기호 업데이터 버튼, 사운드파일경로 업데이트 버튼, 사운드 파일 생성 버튼을 각각 분리하여 출력한다.

단어 오탈자 점검 체크버튼, 의미 업데이트 버튼,  발음기호 업데이트 체크버튼, 사운드파일 경로 업데이트 체크버튼, 사운드 파일 생성 체크버튼을 각각 분리하여 선택 가능하게 하고, 프로세스 버튼은 하나만 출력한다. 버튼을 클릭하면 선택된 동작만 수행한다. 수행 후 업데이트 된 엑셀 파일은 기존 파일명의 뒤에 _update를 붙여서 저장한다.

품사 업데이트 체크박스도를 발음기호 체크박스 다음에 추가하고 관련 기능도 추가한다.

'뚯'은 단어의 의미를 한국어로 표현한다.

현재 파일업로더는 (단어 파일 업로더)로 분명히 하고, 리딩파일 업로더를 하나 더 추가한다. 리딩파일은 엑셀 파일이다. E 열이 비어있으면,  D열의 영어 문장을 한글로 해석하여 E열에 추가한다. 

리딩파일처리 탭에서 파일업로더 위에 파일 템플릿을 다운받을 수 있게 한다. 엑셀 파일 형식이고 2 개의 워크시트로 구성된다. 첫 번재 워크시트는 '본문', 두 번째 워크시트는 '단어'이다. '본문'의 컬럼은 
READING_BOOK_NM,	READING_UNIT_NM,	READING_SENTENCE_SEQ,	READING_SENTENCE,	READING_SETENCE_MEANING,	READING_SENTENCE_SOUND_FILE 이다. '단어'의 컬럼은 READING_BOOK_NM,	READING_UNIT_NM,	READING_VOCA,	READING_VOCA_MEANING,	READING_VOCA_SOUND_FILE 이다
	
리딩파일처리에서 '본문' 워크시트는 E 열이 비어있으면,  D열의 영어 문장을 한글로 해석하여 E열에 추가한다. '단어'워크시트는 D 열이 비어있으면 C 열에 입력된 단어의 한글 뜻을 D 열에 추가한다. 


리딩파일처리에서 '본문' 워크시트는 E 열이 비어있지 않으면 해당 내용을 변경하지 않는다.

리딩파일처리에서 파일을 업로드 하면, 미리보기는 엑셀 파일에서 읽어온 데이터의 tail의 5개 컬럼에 대해 보여주도록 변경한다.

리딩 스크립트 업로드 탭을 추가한다. 워드 파일을 업로드 할 수 있게 하고, 업로드 파일을 문장별로 끊어서 출력할 엑셀파일 D 열에 입력한다. 문단이 바뀔 때는 D 열의 공백 행을 추가한다. A열은 입력된 워드파일의 파일명(확장자제외)를 입력하고, B 열은 Unit 으로 시작되는 워드 문서 내의 단원명을 기재한다. C 열은 문장번호를 기재한다. D열이 공백인 경우도 문장번호는 기재한다. E 열은 D 열에 대한 한글 해석을 추가한다. F 열은 사운드파일의  경로를 기재한다.
C열의 문장번호는 단원별로 새로 시작하도록 한다.

리딩 단원 스크립트 추가 탭을 생성한다. 교재제목, 단원제목과 본문을 입력할 수 있는 텍스트 상자를 출력한다. 본문은 멀티라인 텍스트 입력상자로 출력한다. 
입력완료 버튼을 추가하면 교재제목을 A 열, 단원제목에 B 열을 추가하고, D 열에 본문을 문장별로 끊어서 저장한다. C 열은 문장번호, E 열에는 해석 내용을 저장한다.

[리딩파일처리] 리딩파일 템플릿에 세 번째 워크시트를 추가한다. 워크시트 명은 links이고, 컬럼은 title, link, teacher_only 이다.


리스닝 스크립트 업로드, 리스닝 파일 처리, 리스닝 단원 스크립트 추가 탭을 추가한다.
[리스닝 스크립트 업로드] 워드 파일을 업로드하면 엑셀 파일로 변환 처리한다. Unit으로 시작하는 문단은 단원명으로 인식합하고, 대화 형식을 TOPIC의 경우 화자의 이야기 별로 하나의 sentence 로 처리한다. 한명이 여러 문장을 이야기 한 경우, 각 문장을 별도의 sentence 로 처리한다. 문단이 바뀔 때는 D 열의 공백 행을 추가한다. A열은 입력된 워드파일의 파일명(확장자제외)를 입력하고, B 열은 Unit 으로 시작되는 워드 문서 내의 단원명을 기재한다. C 열은 문장번호를 기재한다. D열이 공백인 경우도 문장번호는 기재한다. E 열은 D 열에 대한 한글 해석을 추가한다. F 열은 사운드파일의  경로를 기재한다.C열의 문장번호는 단원별로 새로 시작하도록 한다.
[리스닝 파일 처리] 리딩파일처리와 같은 형태의 템플릿 엑셀 파일을 다운로드 할 수 있게 한다. 리스닝 파일 업로드를 하면, E열이 비어있을 경우 D 열의 영문을 해석하여 E열을 채운다. 단 "Sally:" 와 같이 콜론으로 시작하는 텍스트는 화자를 의미하므로, 해석하지 않고 그대로 둔다.
[리스닝 단원 스크립트] 리딩단원스크립트와 같은 형식의 UI를 출력한다. 단 스크립트의 처리는 리스닝 스크립트 업로드와 같은 방법으로 처리되어야 한다.


[리스닝 단원 스크립트 추가] 화자를 입력할 수 있는 텍스트 입력박스를 출력한다. 화자는 여러명일 수 있으므로 텍스트입력박스를 추가해서 입력할 수 있게 한다. 처리할 때, 문장의 처음에 나오는 단어가 화자인 경우, "화자:" 형태로 변경하여 처리한다.

[리스닝 단원 스크립트 추가] 대화형식의 스크립트에서는 문단바꿈(공백으로 구분된 줄)을 하나의 문장으로 처리하지 않는다. 

[리스닝 단원 스크립트 추가] Topic 다음에는 한 중의 공백이 필요하다. 또한 한 화자가 여러 문장을 이야기 할 때(토픽 내에서 혼자 이야기 하는 상황)는 문단으로 줄바꿈이 필요하다.
